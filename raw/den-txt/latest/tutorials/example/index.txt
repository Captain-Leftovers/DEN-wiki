Template: Example

   Source

   templates/example

   The example template demonstrates Den’s advanced features: namespaces,
   angle brackets, cross-platform hosts, mutual providers, and custom
   aspect libraries.

Initialize

   Section titled “Initialize”
   Terminal window

   mkdir my-nix && cd my-nix

   nix flake init -t github:vic/den#example
   nix flake update den
   (BUTTON)

Key Features Demonstrated

   Section titled “Key Features Demonstrated”

Cross-Platform Hosts

   Section titled “Cross-Platform Hosts”
   modules/den.nix

   {

   den.hosts.x86_64-linux.igloo.users.alice = { };
   den.hosts.aarch64-darwin.apple.users.alice = { };
   den.homes.x86_64-linux.alice = { };
   }
   (BUTTON)

   One user (alice) across a NixOS host, a Darwin host, and a standalone
   Home-Manager config. The same aspects produce appropriate configs for
   each platform.

Namespaces

   Section titled “Namespaces”
   modules/namespace.nix

   { inputs, den, ... }:

   {
   imports = [ (inputs.den.namespace "eg" true) ];
   _module.args.__findFile = den.lib.__findFile;
   }
   (BUTTON)

   Creates a local namespace eg accessible as a module argument. The true
   flag exposes it as a flake output (flake.denful.eg). Angle brackets are
   enabled via __findFile.

Namespace Aspects

   Section titled “Namespace Aspects”

   The eg/ directory defines reusable aspects under the eg namespace:
   modules/aspects/eg/vm.nix

   { eg, ... }:

   {
   eg.vm.provides = {
   gui.includes = [ eg.vm eg.vm-bootable._.gui eg.xfce-desktop ];
   tui.includes = [ eg.vm eg.vm-bootable._.tui ];
   };
   }
   (BUTTON)

   Aspects can have nested provides — eg.vm._.gui and eg.vm._.tui are
   sub-aspects accessed via the _. provider syntax.

Angle Brackets

   Section titled “Angle Brackets”
   modules/aspects/defaults.nix

   { den, __findFile ? __findFile, ... }:

   {
   den.default.includes = [
   <eg/routes> # resolves to eg.routes
   <den/define-user> # resolves to den.provides.define-user
   ];
   }
   (BUTTON)

   The <name> syntax is shorthand for aspect lookup. <den/define-user>
   resolves to den.provides.define-user. See Angle Brackets.

Mutual Providers

   Section titled “Mutual Providers”

   # modules/aspects/alice.nix — user provides config TO the host

   den.aspects.alice = {
   provides.igloo = { host, ... }: {
   nixos.programs.nh.enable = host.name == "igloo";
   };
   };
   # modules/aspects/igloo.nix — host provides config TO the user
   den.aspects.igloo = {
   provides.alice = { user, ... }: {
   homeManager.programs.helix.enable = user.name == "alice";
   };
   };
   (BUTTON)

   Hosts and users can contribute configuration to each other through
   provides. Alice enables nh on igloo, and igloo enables helix for alice.
   These are cross-providers activated during context transformation.

Tests

   Section titled “Tests”

   # modules/tests.nix — verifies the template works

   {
   checks."alice enabled igloo nh" = checkCond "..."
   igloo.programs.nh.enable;
   checks."igloo enabled alice helix" = checkCond "..."
   alice-at-igloo.programs.helix.enable;
   }
   (BUTTON)

Data Flow

   Section titled “Data Flow”

graph TD
  D["den.nix: igloo/apple + alice"] --> CH["ctx.host"]
  CH --> AI["aspects.igloo<br/>nixos + provides.alice"]
  CH --> CU["ctx.user"]
  CU --> AA["aspects.alice<br/>includes + provides.igloo"]
  AA -->|"routes: mutual"| AI
  AI -->|"routes: mutual"| AA
  CH --> HM["ctx.hm-host"]
  AI --> NixOS["nixosConfigurations.igloo"]
  AA --> NixOS
  AI --> Darwin["darwinConfigurations.apple"]
  AA --> Darwin
